Краткая последовательность логики программ:
===== DK Report Engine =====
DK Engine - модуль, который отвечает за обмен данными между магазином и Головной Компанией (ГК)
 Все действия модуля выполняются в бесконечном цикле.
	1. Основной цикл (бесконечный, интервал 2.5 минуты):
		Попытка подключения к FTP.
		Если попытка удалась:
			Получаем товары и цены с FTP
			Отправляем реализацию
			Отправляем отчет о загруженных в POS файлах
			Переотправляем файлы Z-Отчетов по запросу
			Отправляем лог и БД по запросу
			Отправляем файл БД транзакций по запросу.
			Качаем и запускаем сторонний скрипт (если есть)
			Отключаемся
		 Если не удалась:
			Пробуем загрузить поступления из локальной папки обновления (если есть)
	
	2. Цикл обмена с mini-POS (бесконечный, интервал 5 секунд):
		Забираем файл pos.rep от mini-POS и упаковываем в архив с уникальным номерным именем. Делаем пометку в БД. (по очереди от 1 до 4)
		Выкладываем demo.spr всем mini-POS (не выкладываем, если POS не забрал предыдущий)
		Забираем файл БД транзакций с POS и упаковываем в архив с уникальным номерным именем. Делаем пометку в БД.
	
	3. Цикл выкладывания load.flz (бесконечный, интервал 2 минуты):
		Если файла load.flz нет, выкладываем в папку pos0i, где i - номер очередной папки для mini-POS.
		
	4. Цикл аварийного завершения работы (бесконечный, интервал 1 секунда):
		Если в папке программы есть файл terminate.txt:
			Даем команду на остановку всех циклов
			Ждем 5 секунд
			Удаляем terminate.txt
			Останавливаем программу
**************************************************
===== DK Report Updater =====
	Основной цикл (бесконечный, интервал 10 секунд):
		Если есть файл terminateu.txt:
			Удаляем файл terminateu.txt
			Завершаем программу
		Подключаемся к FTP
		Если подключились:
			Проверяем наличие обновлений на FTP
			Если есть:
				Скачиваем обновление
		Отключаемся
		Если скачали обновление:
			Завершаем работу модуля DK Engine, DK Interface
			Если есть обновление самого модуля обновлений:
				Завершаем работу ВСЕХ модулей
				запускаем сторонний скрипт, который обновит модуль обновлений
			Обновляем модули
			Запускаем модули
**************************************************
===== DK Report Interface =====	
	Реагируем на пользователя.
	Основной цикл:
	Обновляем состояние статистики по обмену.
	
	Отдельный поток: Основной цикл (бесконечный, интервал 10 секунд)
		Проверяем наличие сетевых подключений
		Обновляем статистику подключений
